# Calibration

The scale must be calibrated once after assembly to convert raw ADC counts into pounds. The calibration tool runs directly on the kiosk display — no SSH, keyboard, or terminal needed. All interaction is through the maintenance button on GPIO 18.

The result is saved to `/home/oas/celestial_scale/calibration.json` and loaded automatically by `celestial_scale.py` on its next startup.

## What you need

- A known reference weight (a dumbbell, a bag of dog food with a printed weight, etc.)
- Access to the maintenance button (GPIO 18, Physical Pin 12)

## Launching calibration

### Option A — Button shortcut (recommended)

While the kiosk is running, press the maintenance button **5 times within 3 seconds**. The screen will briefly show "ENTERING CALIBRATION…" and the calibration tool will launch automatically. The kiosk service restarts with the new values when calibration completes.

### Option B — Manual launch via SSH

Stop the service, then run the tool directly:

```bash
sudo systemctl stop celestial-scale-hx711   # or celestial-scale-nau7802
cd /home/oas/celestial_scale
python3 calibrate.py --adc hx711            # or --adc nau7802
```

## Procedure

The calibration tool walks through 5 steps, each shown on the kiosk display.

### Step 1 — Remove weight

The screen shows: **REMOVE ALL WEIGHT / PRESS BUTTON WHEN READY**

Clear the scale platform. Press the button (short or long press) to continue.

### Step 2 — Zero capture

The screen shows: **CAPTURING ZERO READING…** with a progress bar.

Stand clear — the tool automatically captures 20 baseline readings. The bar fills on its own; no button press needed.

### Step 3 — Place weight

The screen shows: **PLACE KNOWN WEIGHT ON SCALE / PRESS BUTTON WHEN READY**

Set your reference weight on the platform. Press the button when ready.

### Step 4 — Load capture

The screen shows: **CAPTURING LOADED READING…** with a progress bar.

Hold still — the tool captures 20 loaded readings automatically.

### Step 5 — Enter known weight

The screen shows a 5-digit selector: `XXXX.X` lbs (e.g. `0050.0` for 50 lbs).

Use the button to enter your reference weight:
- **Short press** (< 1 s release): increment the highlighted digit (0 → 1 → … → 9 → 0)
- **Long press** (≥ 1 s release): advance to the next digit position
- **Long press on the last digit**: confirm the entry and compute the calibration factor

### Completion

On success the screen shows the computed calibration factor and counts down 5 seconds before exiting. If launched via the button shortcut, `systemd` restarts the kiosk automatically and picks up the new values.

On error, the screen shows a message and exits after 10 seconds. Check wiring and try again.

## Calibration file

`calibrate.py` writes `/home/oas/celestial_scale/calibration.json`:

```json
{
  "_note": "Generated by calibrate.py. Run calibrate.py to recalibrate.",
  "zero_offset": 123456,
  "calibration_factor": 420.0000
}
```

`celestial_scale.py` reads this file at startup. If the file is missing or unreadable it falls back to defaults and logs a warning — check the journal if readings seem off:

```bash
journalctl -u celestial-scale-hx711 -n 50
```

## Re-calibrating

Run the calibration tool again at any time — it overwrites the JSON with new values. Useful after:
- Replacing load cells or the HX711/NAU7802 board
- Moving the scale to a new location
- Noticing consistent offset in the displayed weights

## Troubleshooting

- **Progress bar never fills:** No ADC readings received. Check wiring and that `pigpiod` is running (`systemctl status pigpiod`) for HX711, or that I2C is enabled for NAU7802.
- **Inconsistent readings:** Ensure the scale is on a flat, hard surface and the platform is not touching anything on the sides.
- **Negative weights:** The A+ and A- wires on the HX711 (or differential inputs on the NAU7802) are swapped.
- **"No weight detected":** The net raw value between zero and loaded capture was zero — the ADC isn't seeing a difference. Check cell wiring and the combiner board connections.
- **Verbose logging during calibration:**
  ```bash
  python3 /home/oas/celestial_scale/calibrate.py --adc hx711 --log-level DEBUG
  ```
  Debug output includes the mean, standard deviation, and sample count for both the zero and loaded captures.
